# 需求文档

## 简介

本项目旨在开发一个裁判文书网（https://wenshu.court.gov.cn/）的MCP（Model Context Protocol）服务器，使AI助手能够查询、检索和分析中国法院公开的裁判文书数据。

**重要说明**: 裁判文书网需要用户通过支付宝扫码登录才能访问数据。

## 术语表

- **MCP**: Model Context Protocol，一种允许AI模型与外部工具和数据源交互的协议
- **文书服务器**: 裁判文书MCP服务器，提供文书查询和检索功能
- **裁判文书**: 包括判决书、裁定书、调解书等法律文书
- **案件类型**: 如刑事、民事、行政、赔偿、执行等
- **法院级别**: 如最高人民法院、高级人民法院、中级人民法院、基层人民法院

## 需求列表

### 需求1: 文书搜索

**用户故事:** 作为法律研究人员，我希望通过关键词搜索裁判文书，以便找到与我研究相关的法律案例。

#### 验收标准

1. 当用户提供搜索关键词时，文书服务器应返回匹配的裁判文书列表
2. 当返回搜索结果时，文书服务器应包含每个结果的案件名称、案号、法院名称和裁判日期
3. 当没有找到结果时，文书服务器应返回空列表并附带描述性消息

### 需求2: 高级筛选

**用户故事:** 作为律师，我希望按案件类型、法院级别和日期范围筛选裁判文书，以便缩小搜索范围到特定类别。

#### 验收标准

1. 当用户指定案件类型筛选时，文书服务器应仅返回匹配该案件类型的文书
2. 当用户指定法院级别筛选时，文书服务器应仅返回该级别法院的文书
3. 当用户指定日期范围时，文书服务器应仅返回该日期范围内的文书
4. 当组合多个筛选条件时，文书服务器应使用AND逻辑应用所有筛选条件

### 需求3: 文书详情获取

**用户故事:** 作为用户，我希望获取特定裁判文书的完整内容，以便阅读完整的判决书。

#### 验收标准

1. 当用户提供有效的案号或文书ID时，文书服务器应返回完整的文书内容
2. 当返回文书内容时，文书服务器应包含结构化的元数据（当事人、法院、法官、日期等）
3. 如果提供了无效的文书ID，文书服务器应返回适当的错误消息

### 需求4: 案件类型列表

**用户故事:** 作为用户，我希望查看可用的案件类型和法院级别，以便了解筛选选项。

#### 验收标准

1. 文书服务器应提供列出所有可用案件类型的工具
2. 文书服务器应提供列出所有可用法院级别的工具
3. 当列出选项时，文书服务器应同时包含中文名称和代码

### 需求5: 分页支持

**用户故事:** 作为用户，我希望能够翻页浏览搜索结果，以便高效地浏览大量结果集。

#### 验收标准

1. 当搜索结果超过页面限制时，文书服务器应支持分页参数
2. 当返回分页结果时，文书服务器应包含总数和当前页信息
3. 文书服务器应允许配置页面大小，并提供合理的默认值

### 需求6: 错误处理

**用户故事:** 作为用户，我希望在出现问题时获得清晰的错误消息，以便理解和解决问题。

#### 验收标准

1. 如果外部API不可用，文书服务器应返回服务不可用错误并提供重试指导
2. 如果遇到请求限流，文书服务器应返回适当的限流信息
3. 如果提供了无效参数，文书服务器应返回验证错误详情

### 需求7: 支付宝扫码登录

**用户故事:** 作为用户，我希望通过支付宝扫码进行身份验证，以便访问需要登录的裁判文书。

#### 验收标准

1. 文书服务器应提供检查当前登录状态的工具
2. 当用户未登录时，文书服务器应返回支付宝认证的二维码URL
3. 当用户扫码完成认证后，文书服务器应将认证Token存储到本地
4. 当存储的Token存在且未过期时，文书服务器应将其用于后续请求
5. 如果存储的Token已过期，文书服务器应提示重新认证并提供新的二维码
